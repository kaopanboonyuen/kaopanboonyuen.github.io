DE101-PD04 ‚Äî GroupBy, Aggregation & Joins (Analytics & Interview Mastery)

~75‚Äì90 minutes

üéØ Why This Chapter Is Critical

If Pandas were a weapon:

  • Cleaning = sharpening
  • GroupBy & Join = actual power

Most real-world questions are:

  • ‚ÄúWhich group performs best?‚Äù
  • ‚ÄúHow do metrics change by segment?‚Äù
  • ‚ÄúHow do we combine datasets correctly?‚Äù

üß† GroupBy Mental Model (Most Important Idea)

GroupBy = Split ‚Üí Apply ‚Üí Combine

  1. Split data into groups
  2. Apply aggregation or transformation
  3. Combine results into a new table

This mindset works in:

  • Pandas
  • SQL
  • Spark
  • BigQuery

üìä Example Dataset (E-Commerce Analytics)

import pandas as pd

data = {
    "user_id": [1, 2, 1, 3, 2, 3, 1],
    "country": ["US", "US", "US", "FR", "US", "FR", "US"],
    "revenue": [100, 200, 150, 80, 120, 60, 90],
    "device": ["mobile", "desktop", "mobile", "mobile", "mobile", "desktop", "desktop"]
}

df = pd.DataFrame(data)
df

üì¶ Basic GroupBy

df.groupby("country")["revenue"].sum()

Output

country
FR    140
US    660

This answers:

‚ÄúHow much revenue comes from each country?‚Äù

üî¢ Multiple Aggregations

df.groupby("country")["revenue"].agg(["sum", "mean", "count"])

Used daily in:

  • KPI dashboards
  • Business reports
  • Model evaluation

üß† GroupBy Multiple Columns

df.groupby(["country", "device"])["revenue"].sum()

Now you are thinking like an analyst.

üßÆ Resetting Index (Very Common)

df.groupby("country", as_index=False)["revenue"].sum()

Or:

df.groupby("country")["revenue"].sum().reset_index()

This makes output usable for joins.

üß© .transform() ‚Äî Keep Original Shape

df["country_avg_revenue"] = (
    df.groupby("country")["revenue"].transform("mean")
)

Use when:

  • You need group stats per row
  • Feature engineering for ML

üîÅ .apply() ‚Äî Flexible but Dangerous

df.groupby("country").apply(lambda x: x["revenue"].max() - x["revenue"].min())

‚ö†Ô∏è Powerful but slower. Use only when aggregation cannot express logic.

üß† SQL vs Pandas GroupBy

SQL Pandas
GROUP BY groupby()
SUM sum()
AVG mean()
COUNT count()
SQL vs Pandas GroupBy?

Conceptually identical ‚Äî only syntax differs

üîó Why Joins Matter

Real-world data is never in one table.

You often have:

  • users table
  • orders table
  • events table
  • experiments table

Joins connect the story.

üìä Example Tables

Users

users = pd.DataFrame({
    "user_id": [1, 2, 3],
    "country": ["US", "US", "FR"]
})

Orders

orders = pd.DataFrame({
    "order_id": [101, 102, 103, 104],
    "user_id": [1, 2, 1, 4],
    "amount": [250, 300, 150, 500]
})

üîó Inner Join

pd.merge(users, orders, on="user_id", how="inner")

Keeps only matching rows.

üîó Left Join (Most Common)

pd.merge(users, orders, on="user_id", how="left")

Used when:

  • Users are primary
  • Orders may be missing

üîó Right Join

pd.merge(users, orders, on="user_id", how="right")

Less common but useful for audits.

üîó Outer Join

pd.merge(users, orders, on="user_id", how="outer")

Used for:

  • Data reconciliation
  • Finding missing relationships

üß† Join on Different Column Names

pd.merge(
    users,
    orders,
    left_on="user_id",
    right_on="user_id"
)

In real data, column names rarely match.

‚ö†Ô∏è Join Explosion (Critical Warning)

Many-to-many joins can multiply rows.

users.merge(orders, on="user_id")

Always ask:

‚ÄúWhat is the cardinality?‚Äù

üß™ Validation After Join

assert merged["user_id"].notna().all()

Production engineers always validate joins.

üß† Real-World Case (Google / Meta / OpenAI)

Example:

  • Join user metadata
  • Join experiment assignment
  • Aggregate metrics per variant
  • Decide product direction

Bad join ‚Üí wrong decision ‚Üí $$$ lost

üß† Interview Patterns

Interviewers test:

  • GroupBy logic
  • Correct join type
  • Index reset
  • Avoiding row explosion

üèÅ Summary

  • GroupBy = Split ‚Üí Apply ‚Üí Combine
  • .agg() for metrics
  • .transform() for ML features
  • Joins connect reality
  • Validate after join

If you master this chapter, you can survive most analytics interviews.
