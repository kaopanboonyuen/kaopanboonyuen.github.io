DE101-PD05 ‚Äî Performance & Production Patterns (From Pandas to Scale)

~90‚Äì120 minutes

üéØ Why This Chapter Matters

Most Pandas tutorials stop at:

‚ÄúHere is how it works‚Äù

Real engineers must ask:

‚ÄúWill this survive production?‚Äù

Performance mistakes silently:

  • Waste memory
  • Slow pipelines
  • Break models
  • Crash jobs

This chapter teaches you how professionals think.

üß† Mental Model: Pandas Is Vectorized Python

Pandas is fast only when you let it operate on whole columns.

‚ùå Bad:

for i in range(len(df)):
    df.loc[i, "x"] += 1

‚úÖ Good:

df["x"] += 1

Vectorization = speed + clarity.

üö´ Avoid Python Loops (Unless Necessary)

Loops move work from C ‚Üí Python.

Approach Speed
Python loop ‚ùå slow
.apply() ‚ö†Ô∏è medium
Vectorized ops ‚úÖ fastest

üß™ Example: Conditional Logic

‚ùå Slow:

df["flag"] = df["value"].apply(lambda x: 1 if x > 0 else 0)

‚úÖ Fast:

df["flag"] = (df["value"] > 0).astype(int)

üß† Use the Right Data Types

Wrong dtypes waste memory.

Before

df.info()

Convert strings to categories

df["country"] = df["country"].astype("category")
df["device"] = df["device"].astype("category")

This can reduce memory 10√ó‚Äì100√ó.

üß† Numeric Downcasting

df["age"] = pd.to_numeric(df["age"], downcast="integer")
df["revenue"] = pd.to_numeric(df["revenue"], downcast="float")

Used in large ETL pipelines.

üì¶ Memory Profiling

df.memory_usage(deep=True).sum() / 1024**2

Always measure before optimizing.

üß† Copy vs View (Danger Zone)

df2 = df[df["age"] > 18]
df2["flag"] = 1  # ‚ö†Ô∏è SettingWithCopyWarning

Safer:

df2 = df.loc[df["age"] > 18].copy()
df2["flag"] = 1

Silent bugs happen here.

‚öôÔ∏è Chunk Processing (Large Files)

chunks = pd.read_csv("large.csv", chunksize=100_000)

for chunk in chunks:
    process(chunk)

Used when data barely fits in memory.

üß† Efficient Joins

  • Join on indexed columns
  • Avoid object dtype keys
  • Filter before join
users = users.set_index("user_id")
orders = orders.set_index("user_id")

users.join(orders, how="left")

üß† Sorting Is Expensive

df.sort_values("timestamp")

Sorting is O(n log n) Avoid repeated sorts.

üß† Use query() for Readability

df.query("country == 'US' and revenue > 100")

Often faster and clearer than chained indexing.

üß™ Profiling Runtime

%timeit df["revenue"].sum()

In production:

  • Measure
  • Compare
  • Decide

üß† Parallelism: Pandas Is Mostly Single-Threaded

Workarounds:

  • Split data
  • Use multiprocessing
  • Or switch tools

üöÄ When Pandas Is Not Enough

When does Pandas become slow?

When data no longer fits in memory.

At this point, professionals move to:

Tool When
Dask Larger-than-memory Pandas
Spark Distributed clusters
DuckDB Fast SQL analytics
Polars Modern fast DataFrame

üß† Google / Meta / OpenAI Pattern

  1. Prototype in Pandas
  2. Validate logic
  3. Optimize
  4. Scale to Spark / SQL
  5. Monitor in production

Same logic, different engine.

üß™ Real Interview Question

‚ÄúYour Pandas pipeline is slow. What do you do?‚Äù

Expected answer:

  • Measure memory
  • Check dtypes
  • Remove loops
  • Vectorize
  • Reduce joins
  • Scale tool if needed

üèÅ Golden Rules

  • Measure before optimizing
  • Avoid Python loops
  • Control memory
  • Validate joins
  • Think about scale early

üöÄ Final Thought

Pandas is not slow. Bad usage is slow.

Engineers who understand performance write code that survives real systems.
