CS101-LC03 ‚Äî Python for Interviews (From Zero to Thinking Like an Engineer)

üéØ Why Python Matters in Interviews (Real World Edition)

Python is not tested because it is easy.

Python is tested because:

It exposes how you think.

Interviewers look for:

  • Can you break a problem into steps?
  • Do you choose the right container?
  • Can you read your own code after 6 months?
  • Do you understand cost (time & memory)?

Python is a mirror of your thinking quality.

üß† How Strong Python Engineers Think

Before writing code, they ask:

  1. What is the input shape?
  2. What must I track?
  3. What repeats? (loops)
  4. What must be fast lookup? (set / dict)
  5. What logic can become a function?

This chapter builds that instinct.

üîÅ Absolute Basics (Interview Warm-Up)

These problems look easy, but many candidates fail them.

üî¢ Problem 1 ‚Äî Count Prime Numbers (Classic Warm-Up)

Task:
Given n, count how many prime numbers are < n.

def count_primes(n):
    def is_prime(x):
        if x < 2:
            return False
        for i in range(2, int(x**0.5) + 1):
            if x % i == 0:
                return False
        return True

    count = 0
    for i in range(n):
        if is_prime(i):
            count += 1
    return count

üß† Why interviewers love this:

  • Loop control
  • Early break
  • Math intuition
  • Nested functions

üî¢ Problem 2 ‚Äî Frequency Counter (Real Life)

Task: Count how many times each number appears.

def frequency(arr):
    freq = {}
    for x in arr:
        freq[x] = freq.get(x, 0) + 1
    return freq

Used everywhere:

  • Logs
  • NLP tokens
  • Event analytics
  • ML preprocessing

üîÅ Iteration Patterns (Fundamental Thinking)

Looping Over Values

for x in arr:
    print(x)

Use when:

  • You don‚Äôt care about position
  • Logic depends only on value

Looping With Index (IMPORTANT)

for i, x in enumerate(arr):
    print(i, x)

Use when:

  • Position matters
  • Sliding window
  • Adjacent comparison

Looping With Range (Indexing Power)

for i in range(0, len(arr), 2):
    print(arr[i])

This is how real indexing problems are solved.

üîç Searching Patterns

Linear Search (Baseline)

def linear_search(arr, target):
    for x in arr:
        if x == target:
            return True
    return False

‚è±Ô∏è O(n)

Binary Search (Mindset Shift)

def binary_search(arr, target):
    l, r = 0, len(arr) - 1

    while l <= r:
        mid = (l + r) // 2
        if arr[mid] == target:
            return True
        elif arr[mid] < target:
            l = mid + 1
        else:
            r = mid - 1

    return False

‚è±Ô∏è O(log n)

Interviewers care more about:

Why binary search works, not syntax.

üßÆ Vector Thinking (AI & Data Foundation)

Vector Addition

def add_vectors(a, b):
    return [a[i] + b[i] for i in range(len(a))]

Dot Product

def dot(a, b):
    return sum(a[i] * b[i] for i in range(len(a)))

Used in:

  • Neural networks
  • Similarity search
  • Attention mechanisms

üß± Matrix Thinking (2D Indexing)

matrix = [
    [1, 2, 3],
    [4, 5, 6]
]

Traversal

for i in range(len(matrix)):
    for j in range(len(matrix[0])):
        print(matrix[i][j])

Index control = engineering maturity.

Matrix Multiplication (Classic Filter)

def matmul(A, B):
    result = [[0]*len(B[0]) for _ in range(len(A))]

    for i in range(len(A)):
        for j in range(len(B[0])):
            for k in range(len(B)):
                result[i][j] += A[i][k] * B[k][j]

    return result

‚è±Ô∏è O(n¬≥)

üß† Functions = Thinking Units

def square(x: int) -> int:
    return x * x

Good functions:

  • Do one thing
  • Have clear inputs/outputs
  • Are testable

üß± Classes (Engineer Level)

class Vector:
    def __init__(self, values):
        self.values = values

    def dot(self, other):
        return sum(a*b for a, b in zip(self.values, other.values))

Why classes matter:

  • Modeling real systems
  • Clean APIs
  • Interview design questions

üß¨ Inheritance (Abstraction Skill)

class Matrix(Vector):
    def shape(self):
        return (len(self.values), len(self.values[0]))

Used in:

  • Frameworks
  • ML pipelines
  • Systems design

any / all

all(x > 0 for x in arr)
